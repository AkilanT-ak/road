# RouteWatch — Real-Time Accident Risk Predictor

A Flask app that estimates accident risk for a route based on live speed and
traffic congestion, shown on a dashboard-style gauge.

## What changed from the original version

- **Voice warning moved to the browser.** `pyttsx3` needs a local audio
  device and doesn't work on a server like Render, so the "High risk"
  warning is now spoken client-side with the Web Speech API instead.
- **Google Maps API key is read from an environment variable**
  (`GOOGLE_API_KEY`) instead of being hard-coded, so it's never committed
  to the repo. If it's not set, the app still works — the map preview is
  just hidden and traffic level falls back to a neutral estimate.
- **`accident_model.pkl` is generated by `train_model.py`** on synthetic
  data so the app runs immediately, but the data isn't random — it's
  generated from **Nilsson's Power Model**, the published road-safety
  finding that injury risk scales roughly with `(speed / reference
  speed) ^ 4`, plus a secondary congestion effect. It's still a
  stand-in, not a fit to real crash records. Swap in your own model any
  time — it just needs to expose `.predict([[speed, traffic_level]])`
  returning `"Low"` / `"Medium"` / `"High"`.
- Redesigned front end: dashboard/instrument-cluster look with an animated
  risk gauge, digital speed/traffic readouts, and a route map panel.
- Added `gunicorn`, `Procfile`, and `render.yaml` for a clean Render deploy,
  plus a `/health` endpoint.

## Run locally

```bash
pip install -r requirements.txt
python train_model.py        # only needed if accident_model.pkl is missing
export GOOGLE_API_KEY=your_key_here   # optional, for the map + live traffic
python app.py
```

Open http://127.0.0.1:5000

## Deploy to Render

1. Push this folder to a GitHub repo.
2. On Render: **New → Web Service**, connect the repo.
3. Build command: `pip install -r requirements.txt && python train_model.py`
4. Start command: `gunicorn app:app`
5. Add an environment variable `GOOGLE_API_KEY` with your Google Maps API
   key (Directions API + Maps Embed API enabled). The app works without
   it too — you just won't get the map preview or live traffic data.
6. Deploy. (If you use `render.yaml`, Render will pick up the build/start
   commands automatically — you'll just be prompted for `GOOGLE_API_KEY`.)

## Using real accident data instead

The bundled model is grounded in published research (Nilsson's Power
Model), but it's still not fit to real crash records. To train on real
data:

1. Get a dataset with speed and traffic/congestion fields, e.g. the UK
   government's STATS19 road safety data (free, no login):
   https://www.data.gov.uk/dataset/cb7ae6f0-4be6-4935-9277-47e5ce24a11f/road-accidents-safety-data
   (use the `Speed_limit` and `Accident_Severity` columns)
2. Load it with pandas, engineer a `traffic_level` feature if you don't
   have one directly, and train a classifier the same way
   `train_model.py` does.
3. Save it as `accident_model.pkl` in this folder (same interface:
   `.predict([[speed, traffic_level]])` → `"Low"`/`"Medium"`/`"High"`).

## Notes

- The bundled model is a placeholder trained on synthetic-but-research-
  grounded data, meant to make the app runnable end to end with sensible
  behaviour. Replace `accident_model.pkl` with a model trained on real
  data for genuinely accurate predictions.
- On desktop browsers there's usually no real speed sensor, so the app
  simulates a plausible speed reading — this matches the original
  behaviour. On mobile, real GPS speed is used when available.
